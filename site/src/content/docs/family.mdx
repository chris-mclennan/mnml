## How they fit together

They share more than a coat of paint:

- **Shared design language.** Charcoal rounded-square icons, a
  `>` prompt accent in each app's color (cool blue for mnml, gold for
  mixr), monospace wordmarks. Same family settings UI idiom — a
  sectioned scrollable list with `▸ <label>: [active] / other  *`
  rows, the same keybindings (`←→` adjust, `↑↓` move, `r` reset row,
  `R` reset all, `Enter` save, `Esc` cancel).
- **Shared `fim-engine`.** The same in-process FIM (fill-in-the-middle)
  Rust crate powers mnml's optional offline inline suggestions. The
  same quantized model lives in both apps' caches.

You don't need both. mnml is happy in any terminal (iTerm2, Alacritty,
kitty, Terminal.app, Ghostty, Windows Terminal); mixr runs standalone
too. But run them together and the seams disappear.
