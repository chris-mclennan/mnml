# 2026-07-10 hunt summary — 3 SEV-1 · 12 SEV-2 · 6 SEV-3

Five testers back. Per-persona reports staged next to this file.

## SEV-1

1. **api-workflow: `.http` write-back destroys sibling blocks.**
   The 2026-07 "click .http → auto-opens as Request pane"
   path never sets `source_block_name` (`request_pane.rs:550`
   always `None`). Save falls through the `splice_http_block`
   named-block match and OVERWRITES the whole file with a
   single curl — same regression class as
   `http_multi_block_writeback` used to catch, back via a new
   entry point.

2. **api-workflow: Write-back bakes resolved secrets into
   the source.** `send_request_from_active`
   (`src/app/http.rs:3808-3823`) expands `{{VAR}}` in place +
   stores the RESOLVED request in the pane; a later
   `file.save` writes `Bearer devtoken123` where the file had
   `Bearer {{TOKEN}}`. Confirmed via `http.view_source` +
   `http.next_block` + `http.send` sequence.

3. **nvchad: `:` on a Request pane types into the URL bar + Enter fires a network request.**
   Vim users hit `:` all day (`:w`, `:bn`, `:q`). On a
   Request pane it silently corrupts the URL and Enter fires
   the request against the mangled URL. `Esc` doesn't
   revert; session persistence remembers. `src/tui/handlers/pane.rs:2710-2734`.

## SEV-2

4. **api-workflow: `http.copy_ai_prompt` doesn't redact body
   secrets.** Module doc promises header + body redaction;
   only header is implemented. `"apiKey": "sk-live-..."` in
   a body copies to clipboard verbatim.

5. **vscode-mouse: Bitbucket palette-bar chip missing**
   despite `in_palette_bar = true && enabled = true` in
   `~/.config/mnml/integrations/bitbucket.toml`. Only Browser
   renders on row 0. `integrations.refresh` doesn't fix it.
   Something in `src/ui/mod.rs:2056-2071` is short-circuiting.

6. **vscode-mouse: Split-strip hit-rects overlap between
   narrow neighbouring panes.** `split_strip:3:Vertical` and
   `split_strip_term:4` both register at (105, 1) w=3 with
   w=5/w=10 leaves. The `$` / `⊟` / `⊞` core cluster paints
   over the previous leaf's territory.

7. **vscode-keyboard: `set_activity_section` doesn't reset
   focus when re-clicking the same section.** The
   `active_section != section` gate lets Enter-on-a-hit +
   click-back-to-Todos leave focus on the editor — `/` types
   into the file. `src/app/layout.rs:1882-1905`.

8. **vscode-keyboard: New `Copy id` / `Show manifest…` menu
   items have no keyboard reach.** Shift+F10 requires a
   `hover_chip < 2s` (mouse-only). No palette id.
   `src/app/context_menus.rs:379-386`.

9. **vscode-keyboard: Integrations + Agents panels lack
   `<section>_panel_cursor_*` methods.** j/k in Integrations
   silently moves the hidden tree cursor via fall-through.

10. **nvchad: `.md` MdPreview traps every vim ex key.** `:`,
    `/`, `i`, `:e`, `:bn`, `:q` all inert (mode:"none").
    `markdown.edit_raw` exists but has `keys: &[]` — only
    palette (`Ctrl+Shift+P`) reaches it.

11. **nvchad: `?<pattern><CR>` searches for the literal
    string `?<pattern>` instead of reverse-search.** The `?`
    becomes part of the pattern instead of UI chrome.

12. **design: `+ New X` chip has three unrelated visual
    treatments.** Agents = filled green pill, Notes/Integrations
    = plain green text, Sessions = dim-gray text (barely
    reads as clickable). Same gesture, three affordance
    strengths.

13. **design: Sessions keyboard-cursor indicator can vanish
    entirely.** 1-cell left-edge stripe, user-set
    `accent_color` on any session silently overrides
    cursor's cyan (`sessions_panel.rs:182-202`). TODOs/Notes
    use whole-row `bg2` — unmissable.

14. **design: INTEGRATIONS header uses `t.fg` where
    TODOS/NOTES/SESSIONS/AGENTS/CLOUD AGENTS all use
    `t.comment`.** Lone outlier.

15. **design: Integration context menu buries Copy id / Show
    manifest** 6 items deep, right before Remove. Frequently-
    used inspection actions sit below rarely-used reorders.
